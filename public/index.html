This tool only performs passive, read-only checks (headers, TLS, cookies, static markup) against domains hardcoded in server/config/allowlist.js on the server. There is no free-text URL field and no self-service way to add a domain from this page — scanning an arbitrary site is architecturally not possible from here.

To audit a new site, the operator adds its hostname to the allowlist file and redeploys. That's the same authorization model self-verification tools like Google Search Console use, simplified to a static list since this project only audits sites its own author controls.
